# Obsidian Note Templates — NotebookLM Imports

Use these templates exactly when writing NotebookLM output into the vault.
All imports land in `60-NotebookLM/` as `status: draft` first.
File naming: `nlm-[type]-[topic-in-kebab-case]-[YYYY-MM-DD].md`

---

## Processing Checklist

Run this after every import before changing status from `draft`:

```
- [ ] Front matter complete (title, date, tags, status: draft, source, notebook_id)
- [ ] Summary written in own words (not raw API output)
- [ ] Key insights extracted as atomic bullet points
- [ ] Citations mapped to 30-Resources/ source notes
- [ ] Contradictions with vault notes flagged in ## Claude's Notes
- [ ] [[wikilinks]] added to all recognized concepts
- [ ] Next actions listed: which insights to atomize, which MOCs to update
- [ ] File naming follows: nlm-[type]-[topic]-[YYYY-MM-DD].md
```

Only change `status: draft` → `status: in-progress` once this checklist is complete.
Only change to `status: evergreen` once insights are atomized and fully linked.
